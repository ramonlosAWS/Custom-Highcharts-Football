A) Current: Scatter + Fixed Divs (680px widget)

Each button is a separate data point, centered on X coordinate. Div width must match widget width / 4.

680px widget → 170px divs
650px widget → 163px divs (current config)
750px widget → 163px divs (WRONG - gaps appear)
Problem: If you resize the visual, the gaps break. Each width needs a different div width.

B) Flexbox: Single Data Point (auto-adapts to any width)

One data point with all buttons in a flex container. Buttons auto-fill available width.

680px widget
650px widget (same config, no changes)
750px widget (same config, no changes)
Advantage: Same config works at any width. No div width tuning needed.
Trade-off: triggerClick fires for the whole point, not individual buttons. Need to test if QuickSight can distinguish which button was clicked.

C) Flexbox Hybrid: Individual Series + Flex Styling

Separate series per button (for click actions), but using flex:1 instead of fixed width.

680px widget
650px widget
750px widget
Still has the X-coordinate alignment issue since each button is a separate point. Flex styling on the div doesn't help because the div position is determined by the X coordinate.

Conclusion

Flexbox (B) is visually perfect at any width, but the click target is one data point — all 4 buttons trigger the same action. In QuickSight, you'd need the map approach with triggerClick passing rowIndex, which should still work since each button is a separate row in the mapped data.

Scatter (A) works perfectly when div width = widget width / numButtons. The formula is simple, just needs to be set once per widget width.

Recommendation: If the FA filter widget width is fixed (won't change), stick with scatter and set divWidth correctly. If it might change, flexbox is more resilient.
